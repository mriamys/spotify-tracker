# 🎵 Spotify New Track Bot

Бот следит за новыми релизами артистов из твоих подписок в Spotify и автоматически добавляет новые треки в указанный плейлист.

---

## Как это работает

### Первый запуск — формирование базы
При первом запуске бот проходит по всем артистам на которых ты подписан и добавляет **по 1 треку** из их последнего релиза. Это нужно чтобы зафиксировать точку отсчёта — что считать "старым", а что "новым".

### Ежедневный мониторинг
Каждый день в `03:00` бот проверяет всех артистов. Если у кого-то вышел новый релиз после последней проверки — добавляет **все треки** из него в плейлист.

### Эффективная работа с лимитами
Бот оптимизирован для минимизации нагрузки на API Spotify:
- **Группировка запросов:** Альбомы и синглы запрашиваются одним пакетом, что снижает количество вызовов API в 2 раза.
- **Умный 429 отклик:** Бот ловит ошибку `429` и ждет столько, сколько просит Spotify. Если же Spotify накладывает «жесткий» лимит (более часа), бот прекращает работу до следующего дня, чтобы не нагружать систему бесконечным ожиданием.
- **Автоматическое обновление токена:** Токен обновляется перед каждым сканированием, предотвращая ошибки доступа при длительных сессиях.
- **Безопасные паузы:** Между запросами к артистам добавлены фиксированные задержки (`SAFE_DELAY`).

## Установка

### 1. Клонируй репозиторий

```bash
git clone https://github.com/твой_ник/spotify-new-track-bot.git
cd spotify-new-track-bot
```

### 2. Создай виртуальное окружение и установи зависимости

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Linux / macOS
source venv/bin/activate

pip install -r requirements.txt
```

### 3. Настрой Spotify Developer App

1. Зайди на [developer.spotify.com/dashboard](https://developer.spotify.com/dashboard)
2. Нажми **Create app**
3. Заполни название и описание (любые)
4. В поле **Redirect URIs** добавь: `http://127.0.0.1:8888/callback`
5. Сохрани и открой **Settings** — там будут `Client ID` и `Client Secret`

### 4. Создай файл `.env`

Скопируй `.env.example` в `.env` и заполни своими данными:

```bash
cp .env.example .env
```

Открой `.env` и вставь свои значения. Подробные инструкции по каждому полю написаны прямо в `.env.example`.

### 5. Запусти бота

```bash
python spotify_bot.py
```

При первом запуске Spotify попросит авторизацию — откроется ссылка в консоли, скопируй её в браузер, разреши доступ, и вставь URL из адресной строки обратно в консоль.

## Структура файлов

```
spotify-new-track-bot/
├── spotify_bot.py      # основной код бота
├── requirements.txt    # зависимости
├── .env.example        # пример конфига (заполни и переименуй в .env)
├── .gitignore
└── README.md
```

После первого запуска появятся ещё два файла — они в `.gitignore` и в репозиторий не попадут:
- `.env` — твои секретные ключи
- `bot_state.json` — сохранённый прогресс бота
- `.cache` — кеш токена Spotify

## Настройки внутри бота

В начале файла `spotify_bot.py` есть несколько параметров которые можно поменять:

```python
RUN_TIME = "03:00"   # время ежедневного запуска
SAFE_DELAY = 3       # пауза между запросами в секундах
```

## Важные моменты

**Не трогай подписки во время первичного сканирования.** Бот идёт по списку артистов по индексу и сохраняет позицию. Если добавить или удалить артиста пока идёт первый проход — позиции сдвинутся и кто-то может пропуститься. После того как `initial_scan_done` стал `true` — можно спокойно менять подписки.

**Оставляй консоль открытой.** Если Spotify даст длинный лимит (`Retry-After: 84000` секунд — такое бывает) — бот будет спать и ждать. Закрывать консоль не нужно, он сам продолжит. Прогресс в любом случае сохранён, и при следующем ручном запуске продолжит с нужного места.

**Один запуск в сутки.** Бот не будет запускаться повторно если уже отработал сегодня. Это защита от случайного превышения лимитов Spotify.
